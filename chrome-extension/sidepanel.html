How the Interceptor Works

The extension monitors all HTTP traffic to localhost, 127.0.0.1, and 0.0.0.0 on every port. Requests appear in real-time in the Interceptor tab with method, URL, status code, and timing information. No configuration needed — it works automatically when the extension is active.

Regex Filtering

Click the .* button in the filter bar to enable regex mode. Patterns are case-insensitive.

  • /api/(users|posts) — match specific endpoints
  • :300[0-9]/ — match ports 3000–3009
  • :([3-9]\d{3})/ — match all ports 3000+
  • \.(json|xml)$ — match by file extension
  • ^http://localhost:8080/api — exact host + path prefix

Port Patterns

Use regex filter to target specific ports or ranges:

  • :8080/ — single port
  • :(3000|5173|8080)/ — specific ports
  • :8\d{3}/ — range 8000–8999
  • :(3\d{3}|4\d{3})/ — range 3000–4999
  • localhost: — all localhost traffic

Proxy Setup

Switch to Custom Proxy mode and enter your proxy URL. All browser traffic routes through it. Supported protocols:

  • http://host:port — HTTP proxy
  • https://host:port — HTTPS proxy
  • socks5://host:port — SOCKS5 tunnel
  • socks4://host:port — SOCKS4 tunnel

Use the CORS Bypass Proxy URL field to set a custom proxy endpoint (e.g. a Vercel Edge Function) that queFork will use to bypass CORS restrictions on API requests.

CORS Bypass via Extension Bridge

When queFork detects this extension on the page, API requests are routed through the extension's service worker which has no CORS restrictions. This happens automatically — no server-side changes or browser flags needed. The footer indicator in queFork shows "Extension bridge" when active.

Maximising Capabilities

Combine the interceptor with regex filters to isolate traffic from a specific service while running multiple dev servers. Use the proxy to route traffic through debugging proxies like Burp Suite or mitmproxy for deep packet inspection. Export intercepted logs as JSON for analysis or sharing with your team.
